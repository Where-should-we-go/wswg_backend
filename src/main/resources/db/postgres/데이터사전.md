## 🧳 여행 & 지도

### trips — 여행 (계획↔기록, 문서)
| 컬럼 | 타입 | 제약 | 역할 |
|------|------|------|------|
| trip_id | BIGSERIAL | PK | 여행 식별자 |
| title | VARCHAR(255) | NN | 제목 |
| start_date | DATE | | 시작일 |
| end_date | DATE | | 종료일. **완료 판정: < 오늘** |
| user_id | BIGINT | FK→users, nullable | 개인 여행(XOR) |
| group_id | BIGINT | FK→groups, nullable | 모임 여행(XOR) |
| **data** | **JSONB** | NN, default '{}', GIN | **🌟 항목·미디어 전체 문서** (아래 구조) |
| created_at | TIMESTAMPTZ | NN | 생성 |
| updated_at | TIMESTAMPTZ | NN, 트리거 | 수정 |
| — | | CHECK XOR / CHECK dates | 개인·모임 택1 / end≥start |

**`data` JSONB 구조** (강제 아님):
```jsonc
{ "items": [
  { "content_id":126508|null, "title":"경복궁", "type":"관광",
    "lat":37.5,"lng":126.9, "visitDate":"...","order":1,"time":"10:00",
    "media":[{"type":"PHOTO","url":"...","metadata":{...}}],
    "properties":{"budget":0,"rating":5,"memo":"..."} }
]}
```
- 항목·미디어·자유필드 전부 여기 (자기 여행 안에서만 쓰임)
- 동시 편집 대상 = data (Redis 문서 ↔ Batch Worker flush)

### group_region_media — 그룹 지도 대표 미디어
| 컬럼 | 타입 | 제약 | 역할 |
|------|------|------|------|
| id | BIGSERIAL | PK | 식별자 |
| group_id | BIGINT | NN, FK→groups | **어느 그룹 지도** |
| trip_id | BIGINT | FK→trips, SET NULL | 출처 여행(선택) |
| sido_code | INTEGER | NN, FK→sidos | 지역 시도 |
| gugun_code | INTEGER | nullable | 지역 구군(미상 허용) |
| media_type | VARCHAR(10) | NN, CHECK(PHOTO/AUDIO/VIDEO) | 미디어 종류 |
| media_url | TEXT | NN | **선택된 대표 미디어 url(복사)** |
| geom | geometry(Point,4326) | nullable, GIST | 핀 좌표(선택) |
| metadata | JSONB | NN, default '{}' | 썸네일/메타(선택) |
| created_at | TIMESTAMPTZ | NN | 생성 |
| — | | **UNIQUE(group_id, sido_code, COALESCE(gugun_code,-1))** | **지역당 1개** |

> 그룹 지도 = 이 테이블만 읽어 지역+대표 추억 렌더. trips.data 미디어 중 골라 url을 복사(스냅샷).

## 📌 필드 설계 핵심 원칙
- **항목/미디어 = `trips.data` JSONB** (자기 여행 안에서 끝남 → 문서). 노션식 자유 필드는 `properties`.
- **그룹 지도 대표 = `group_region_media` 관계형** (group·trip FK + 지역당 1개 UNIQUE).
- **attractions = 정규화 유지**(마스터). trips.data는 content_id만 소프트 참조(FK 아님).
- **JSONB 원칙**: "자기 안에서 끝나는 중첩 데이터"에만. **관계·제약 있으면 테이블**.
- 좌표: lat/lng 입력원 → geom GENERATED(3NF). PK=BIGSERIAL, 시각=TIMESTAMPTZ(단 users.created_at=DATE).
